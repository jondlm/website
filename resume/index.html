Stripe

Oct 2020 - Aug 2024

I was a founding engineer on the JavaScript Infrastructure team at Stripe. I did whatever it took to make JS devs at Stripe write higher quality code, be more productive, and worry less about maintaining their own tooling. Over the course of my four year tenure, I led and contributed to many projects that established JavaScript as a first-class supported language. Here are some highlights:

  • Created a JavaScript CLI tool used daily by ~600 engineers. It managed JS projects integrated with pnpm, Bazel, and our remote execution infrastructure. It enabled a true monorepo setup that previously was not possible at Stripe.
  • Created and maintained a set of centralized JS dependencies used by monorepo projects. Folks could use the shared tooling without worrying about managing their own test runner, linter, code formatter, build tooling, and typechecking. Customer teams got upgrades for free.
  • Migrated our largest JavaScript codebases (over 4 million lines of code) to a new development optimized bundler that sped up initial development times by 20x.
  • Improved our remote execution system (featured in this YouTube presentation) that decreased Bazel memory (20GiB -> 10GiB), sped up JS execution by 10% (in common cases), and improved sandbox setup up to 5x for JS projects. This involved writing low-level Go code that performed syscalls within Firecracker VMs.
